Os Webhooks permitem integração em tempo real entre a Zynx API e o WhatsApp, permitindo sincronização e compartilhamento automatizados de dados.

Esse recurso possibilita a criação de bots de autoatendimento e sistemas multi-serviço com eventos em tempo real.

## Configurando Webhooks por Instância

Cada instância pode ser configurada com seus próprios webhooks de forma independente. Isso permite que você controle exatamente quais eventos deseja receber para cada instância.

### Configurar Webhook

Para configurar um webhook em uma instância específica, use o endpoint `/webhook/set`:

```json
POST /webhook/set/{instance}

{
  "url": "https://seu-dominio.com/webhook",
  "webhook_by_events": false,
  "webhook_base64": false,
  "events": [
    "QRCODE_UPDATED",
    "MESSAGES_UPSERT",
    "MESSAGES_UPDATE",
    "MESSAGES_DELETE",
    "SEND_MESSAGE",
    "CONNECTION_UPDATE"
  ]
}
```

### Parâmetros

| Parâmetro         | Tipo    | Obrigatório | Descrição |
|-------------------|---------|-------------|-----------|
| url               | string  | Sim         | URL para receber os eventos do webhook |
| webhook_by_events | boolean | Não         | Gera uma URL específica para cada evento |
| webhook_base64    | boolean | Não         | Codifica o payload em base64 |
| events            | array   | Não         | Lista de eventos a serem monitorados |

### Obter Configuração do Webhook

Para verificar a configuração atual de webhooks de uma instância:

```
GET /webhook/find/{instance}
```

Resposta:

```json
{
  "enabled": true,
  "url": "https://seu-dominio.com/webhook",
  "webhookByEvents": false,
  "events": [
    "QRCODE_UPDATED",
    "MESSAGES_UPSERT",
    "MESSAGES_UPDATE",
    "MESSAGES_DELETE",
    "SEND_MESSAGE",
    "CONNECTION_UPDATE"
  ]
}
```

## Eventos Suportados

Estes são todos os eventos de webhook disponíveis:

| Evento | Descrição |
|--------|-----------|
| APPLICATION_STARTUP | Notifica quando a aplicação inicia |
| QRCODE_UPDATED | Envia o QR code em base64 para leitura |
| CONNECTION_UPDATE | Informa o status da conexão com WhatsApp |
| MESSAGES_SET | Lista inicial de mensagens carregadas |
| MESSAGES_UPSERT | Nova mensagem recebida |
| MESSAGES_UPDATE | Mensagem foi atualizada |
| MESSAGES_DELETE | Mensagem foi deletada |
| SEND_MESSAGE | Mensagem foi enviada |
| CONTACTS_SET | Lista inicial de contatos |
| CONTACTS_UPSERT | Contatos foram recarregados |
| CONTACTS_UPDATE | Contato foi atualizado |
| PRESENCE_UPDATE | Status de presença do usuário (online, digitando, etc) |
| CHATS_SET | Lista inicial de chats |
| CHATS_UPDATE | Chat foi atualizado |
| CHATS_UPSERT | Novo chat adicionado |
| CHATS_DELETE | Chat foi deletado |
| GROUPS_UPSERT | Novo grupo criado |
| GROUPS_UPDATE | Grupo foi atualizado |
| GROUP_PARTICIPANTS_UPDATE | Participante adicionado, removido ou promovido |
| NEW_TOKEN | Token JWT foi renovado |

## Webhook por Eventos

Quando você ativa `webhook_by_events`, a Zynx adiciona o nome do evento ao final da URL automaticamente.

### Exemplo

Se sua URL é `https://seu-dominio.com/webhook/`, com `webhook_by_events` ativado:

- `QRCODE_UPDATED` → `https://seu-dominio.com/webhook/qrcode-updated`
- `MESSAGES_UPSERT` → `https://seu-dominio.com/webhook/messages-upsert`
- `CONNECTION_UPDATE` → `https://seu-dominio.com/webhook/connection-update`

Isso permite que você tenha endpoints diferentes para cada tipo de evento na mesma URL base.

## Dicas

- Configure webhooks diferentes para cada instância conforme suas necessidades
- Use `webhook_by_events` se precisar de lógica diferente para cada tipo de evento
- Sempre valide que sua URL de webhook está acessível e respondendo com status 200
- Implemente retry logic na sua aplicação para garantir que os eventos sejam processados
